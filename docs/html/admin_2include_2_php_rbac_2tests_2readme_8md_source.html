1 #Instructions for running Unit Tests.
2 
3 ##Front Matter
4 
5 * The Unit Tests should be run using a database specific for Unit Testing. This way your dev/testing/production data will not be affected.
6 * To run the Unit Tests using the MySQL adapter you will need to have an existing database with the proper tables and default data prior to running the tests.
7 * If you are running the Unit Tests for the SQLite adapter the database will be created/overwritten for you automatically.
8 
9 ##The Setup
10 
11 * Create the database and tables
12  * MySQL
13  * Execute the queries located in the 'mysql.sql' file in the 'rbac/PhpRbac/database/' directory
14  * **WARNING:** Make sure you replace 'PREFIX_' appropriately
15  * SQLite
16  * The database will be created/overwritten for you automatically when you run the Unit Tests
17 * Navigate to 'rbac/PhpRbac/tests/database' and open up 'database.config'. Change the database connection info accordingly
18 * Navigate to 'rbac/PhpRbac/tests' and open up 'phpunit_mysql.xml'. Change the database connection info accordingly. Don't forget to change the database name in the DNS string (this is for the DBUnit connection, fixture and datasets)
19 
20 ##Run The Unit Tests
21 
22 * You will need to navigate to 'rbac/PhpRbac/tests/' in order to execute the following commands.
23 
24 ###On Linux
25 
26 **Note:** Make sure you make 'mysql_tests.sh' and 'sqlite_tests.sh' executable
27 
28 * To run the tests for MySQL: ./mysql_tests.sh
29 * To run the tests for SQLite: ./sqlite_tests.sh
30 
31 ###On Windows
32 
33 * To run the tests for MySQL: mysql_tests.bat
34 * To run the tests for SQLite: sqlite_tests.bat
35 
36 ##Notes
37 
38 * Make sure you alter the 'rbac/PhpRbac/tests/database/database.config' file (see above) before switching between MySQL and SQLite tests.
39 * We've created scripts for Windows and Linux (any OS that has sh/bash available). All scripts will:
40  * Execute PHPUnit with the proper xml configuration file.
41  * Accept additional PHPUnit parameters (i.e. --colors).
42  * Pause after tests so they can be run from a GUI directory/file explorer application.
43 * **Thanks to the AuraPHP team for helping us bootstrap our Unit Testing methods**
